# Two-Pass FK-Safe Batch Commit (C# / EF Core / PostgreSQL)

Pattern for ETL where child rows reference a DB-generated parent PK. EF Core populates the parent entity's `.Id` after `SaveChangesAsync()` in Pass 1, making it available for building child entities in Pass 2 — all within the same per-batch transaction.

## Total DB writes per batch

- Pass 1: 1 `SaveChangesAsync` for all headers (bulk)
- Pass 2: 1 `SaveChangesAsync` for all children (bulk)
- 1 `CommitAsync`

Regardless of `batch_size`, total write operations = **3 per batch**.

## When NOT to use

- Children do not have FK dependencies — single-pass is simpler
- Parent IDs are generated by the application (not DB) — no need for two passes
